# CLIENT_TEMPLATE_OS

## Goal
Portable template for cloning a business operations agent system per client.

## Required Client Files
- CLIENT.md
- BRAND.md
- OFFER.md
- SOPS.md
- STACK.md
- LEADS.md
- CONTENT_CALENDAR.md
- PIPELINE.md
- AUTOMATIONS.md
- APPROVALS.md

## Client Variables
- company name
- industry
- timezone
- communication channels
- CRM/tool stack
- brand voice
- approval rules
- reporting cadence

## Template Layers
1. Core operating system
2. Client identity layer
3. Tool/integration layer
4. Memory layer
5. Reporting layer

## Client Success Criteria
- one-click-ish setup checklist
- role-based agent mapping
- isolated brand memory
- isolated approval policy
- reusable recurring workflows
